# Отгрузки — TODO для миграции на бэкенд

> Этот файл содержит список фич, которые сейчас реализованы на localStorage  
> и должны быть перенесены на бэкенд-API в будущем.

## Критичные (Core API)

- [ ] **CRUD отгрузок** — `/shipments` endpoints (GET, POST, PATCH, DELETE)
- [ ] **Сканирование** — `/shipments/:id/items/:itemId` (PATCH для обновления scannedQuantity)
- [ ] **Удаление скана** — `/shipments/:id/items/:itemId/scans/:scanId` (DELETE)
- [ ] **Завершение отгрузки** — `/shipments/:id/complete` (POST) — списание остатков со склада

## Связки и автоматизация

- [ ] **Авто-создание приемки** — при завершении отгрузки на другой склад, автоматически создавать сессию приемки (`ReceivingSession`) с теми же позициями
- [ ] **Статус «В пути»** — пока приемка на складе-получателе не завершена, оборудование считается «в пути»:
  - Остатки списаны со склада-отправителя
  - Но ещё не появились на складе-получателе
  - На странице Остатков (Stock) отображать колонку/бейдж «В пути: X шт»
- [ ] **Связь отгрузка ↔ приемка** — хранить `linkedReceivingId` в отгрузке и `linkedShippingId` в приемке для отслеживания пар

## Интеграция с магазинами

- [ ] **Авто-перенос оборудования** — при отгрузке в магазин, оборудование должно автоматически привязываться к магазину
- [ ] **Заявки из магазинов** — магазины создают заявки на оборудование → заявки превращаются в отгрузки

## Файловый flow

- [ ] **Шаблон заявки** — кнопка «Скачать шаблон» генерирует Excel/CSV с полным каталогом позиций и пустыми полями количества
- [ ] **Загрузка заполненного шаблона** — парсинг файла → автоматическое создание отгрузки с предзаполненными позициями
